# Markmap 语法参考

渲染环境: SVG canvas 1200x800, initialExpandLevel=-1 (全展开), maxInitialScale=2.

## 基本语法

Markmap 使用标准 Markdown 的标题层级和列表缩进构建思维导图树:

- `#` (一级标题) = 根节点
- `##` (二级标题) = 一级分支
- `###` (三级标题) = 二级分支
- 或用列表 `-` 代替标题, 通过缩进控制层级

两种风格可以混用, 但建议统一使用标题层级或统一使用列表.

## 标题层级风格

```markdown
# 项目架构

## 前端
### React 组件库
### 状态管理
### 路由系统

## 后端
### API 服务
### 数据库
### 缓存层

## 基础设施
### CI/CD
### 监控告警
### 日志收集
```

## 列表缩进风格

```markdown
# 学习路线

- 编程基础
  - 数据结构与算法
  - 设计模式
  - 操作系统
- Web 开发
  - HTML/CSS
  - JavaScript
  - 框架
    - React
    - Vue
- 数据库
  - SQL
  - NoSQL
  - 索引优化
```

## 富文本支持

节点内容支持 Markdown 格式:
- **加粗**: `**文本**`
- *斜体*: `*文本*`
- `代码`: 反引号包裹
- [链接]: `[文本](URL)`
- ~~删除线~~: `~~文本~~`

```markdown
# 技术栈选型

## *前端*
- **框架**: `React 18`
- **构建**: `Vite`

## *后端*
- **语言**: `Python`
- **框架**: `FastAPI`
```

## 常见陷阱

1. **缩进必须一致**: 使用相同数量的空格缩进, 不要混用 Tab 和空格
2. **列表层级深度**: 建议不超过 4-5 层, 过深会导致图表难以阅读
3. **根节点**: 第一个 `#` 标题或第一行内容作为根节点, 只能有一个根
4. **特殊字符**: 节点内容避免使用 Markdown 语法冲突字符, 或用反引号包裹
5. **空行影响**: 列表项之间不要加空行, 否则可能打断层级关系
6. **初始展开**: 渲染配置 initialExpandLevel=-1 表示所有节点完全展开, 无需手动控制
