## 四、关键发现

1. **"字节系独家信源"卖点实测不成立**: 豆包 70 条结果仅 2 条字节系(头条剧集榜、抖音百科黄仁勋), 其余全是新华网/人民网/搜狗百科/smzdm/python.org/research.google 等主流权威源. 豆包赢靠**召回精准度 + 权威源覆盖**, 不是字节独家内容.

2. **豆包召回质量(4.21)逼近有总结的 Gemini(4.43)**: 作为"纯召回层"(无总结), 能接近 Gemini 的信息密度, 且**快 20 倍**(2s vs 40s)、**零失败**(14/14 vs 13/14). 性价比极高.

3. **豆包强项符合预期**: 中文生活资讯类(剧集 Q02 / 旅游 Q04 / 数码 Q06 / 菜谱 Q03 / 热梗 Q05)全部胜出, 这是个人助手高频场景.

4. **豆包在"非预期区"也意外能打**: 英文技术(Q07 命中掘金)、编程(Q08 召回 superfastpython 英文权威)、学术(Q09 召回 research.google)、国际新闻(Q11 人民网/新华网). 召回覆盖面全面.

5. **智谱最弱**: 无源 URL(大量 `ref_N` 占位)、混入老内容(2015/2012)、1 次 400 错误. 召回质量明显垫底.

6. **百度独特价值在"实时性"**: Q01 小时报级别的新闻频率, 这是其他三路都没有的. 但标题党多、质量参差.

7. **Gemini 源 URL 是死链 (重要局限)**: 经中转节点走 Vertex AI Search, 所有源 URL 被包成 `vertexaisearch.cloud.google.com/grounding-api-redirect/<ID>` 内部跳转, 公开访问 404, **不可作为可点击引用**. 但 `title` 字段仍携带真实源域名 (csdn.net / python.org / wikipedia.org / nmpa.gov.cn 等), 评估源多样性时已据此判读. Gemini 真正可用的是 `answer` 总结文本. (注: 项目 `grounding_search.py` 同样经此中转, 生产环境 source URL 亦是死链, 只因项目用 answer 不外显 URL 未暴露.)

## 五、结论与建议

**核心问题: 智谱 + 百度能否覆盖? 基本能, 但豆包有明显升级价值.**

- 字节独家信源理由 **不成立**, 别因此引入.
- 但豆包作为**召回源**, 质量全面碾压智谱, 在生活资讯类明显优于百度.
- Gemini 的"总结能力"独一档, 无法被召回层替代.

**推荐方案 (按性价比排序)**:

1. **用豆包替换 zhipu_search**(强烈推荐): 召回质量全面优于智谱, REST 接入同样简单, 免费 500 次/月 + AFP 抵扣, 快且稳. zhipu 的无 URL/老内容/失败问题一并解决.
2. **保留 baidu**: 实时新闻小时报的独特价值不可替代.
3. **保留 gemini**: 总结能力不可替代(它定位是"带搜索的问答", 非纯召回).

即最终搜索栈: **Gemini Grounding(总结问答) + 豆包(主力召回, 替换 zhipu) + 百度(实时新闻)**. 豆包以 REST 外部工具形态接入(避开 stdio MCP).

**若严格遵循"不可替代才引入"**: 豆包不构成不可替代(智谱+百度+gemini 能覆盖), 但作为 zhipu 的**等价升级**(同位置替换, 不增加维护面), 边际成本几乎为零, 收益明显.

## 六、补充 PK: 技术搜索主场 (doubao vs zhipu, 智谱强项)

10 条技术 query (官方文档/具体库/报错/前沿框架/DB/CLI/算法/DevOps/架构/小众深度), 量化两家的数据源覆盖范围. 详见 `tech_pk.md`.

### 信源类型覆盖汇总

| 源类型 | doubao | zhipu |
|--------|--------|-------|
| **官方文档** | **7** | **0** |
| 中文社区 | 32 | 41 |
| 其他(含英文权威) | 10 | 7 |
| 厂商云 | 1 | 1 |
| 无URL | 0 | 1 |

### 关键发现

1. **官方文档 doubao 7 / zhipu 0**: doubao 召回了 pydantic.dev、postgresql.org、docker.com×2、rust-lang.org×2 等官方源; **zhipu 在 10 条技术 query 里一条官方文档都没命中**. 技术问题查官方文档远比 csdn 转载可靠, 这是质量分水岭.
2. **zhipu 高度依赖 csdn 单一源**: 50 条结果里 csdn.net 占绝大头, 源类型单一; doubao 源多元(官方+英文权威 topcoder + 中文社区 + 厂商云).
3. **逐条胜负**: doubao 明显胜 5 条 (t02 Pydantic / t05 PostgreSQL / t07 红黑树 / t08 Docker / t10 Rust, 均靠官方源), 打平 5 条, **zhipu 0 条占优**.
4. **zhipu 无 URL 问题依旧** (t02 出现 1 条 `ref_N` 占位).

### 结论

即使在智谱的传统强项(技术搜索)主场上, doubao 也**全面不输且多数更优**. 数据源覆盖范围上, **doubao 能触达官方文档/英文权威源, zhipu 基本只在中文社区内容池(以 csdn 为主)打转**. 加之 doubao 500 次/月免费 + AFP 抵扣(实际比 zhipu 0.01 元/次按量更省), **用 doubao 替换 zhipu 在技术搜索维度同样成立**.
